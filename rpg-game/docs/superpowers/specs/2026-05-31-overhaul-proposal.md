# デウス・コード 大幅見直し提案書

- 日付: 2026-05-31
- 方法: 5領域(描画/ワールド・マップ/ゲームプレイ/アーキテクチャ/コンテンツ)を並列監査し統合
- 制約: 品質重視(多少重くても可) / AI画像生成キー無し(画像は外部調達) / 当面ビルド無し維持

## 最重要の2発見

1. **歩行が「変わって見えない」真因＝元アートに脚の歩行キーフレームが無い。**
   原図 `kaito_generated_sheet.png` の各方向4フレームは脚ポーズがほぼ不変。だから手続き描画でも切り出し骨格でも「元のまま」では絶対に歩いて見えない。動きは**足す**しかない（juice強化 か 歩行キーフレーム原図を作る か 骨格化）。加えて `generate_walk_sprites.py` が16フレームを個別トリミングしたため方向/フレーム間で身長・接地がバラつき「右向き崩壊」が起きていた。

2. **見た目以前に、RPGとして壊れている客観バグが眠っている（ユーザー未報告）。**
   - セーブが `player` と `currentMap` しか保存せず `loadGame` は未呼び出し → **実質セーブ無効**
   - レベルアップで exp を減算しない → **多段レベルアップ暴発**
   - バフ/支援魔法/状態異常が計算未接続 → **死にメカニクス**
   - ドロップ装備が配列とオブジェクトの不一致で **永久回収不能**
   - `assets/bgm/` が存在せず **全編無音**（エラー握り潰し）
   - 装備が player 専用 → **他パーティが装備不可**

## 残す（良い土台）
UIとワールド描画の完全分離(DOM/canvas)、単一rAFループ、checkCollision集約、findSafeSpawnPoint、データ駆動の良例(敵AI/成長曲線/item-magic-equip DB/storyスキーマ)、webp最適化、BGM世代トークン、2つのオーサリングツール、computeWalkJuiceのパラメータ化。

## 作り直す
1. **歩行キャラの躍動**(最優先): スプライト再整列(generate_walk_sprites.py改修)＋強juice＋整数スナップ。骨格は不採用。
2. **マップ座標系**: worldScale=1.55 二重座標を撤廃→per-map scaleで段階的に worldScale=1 へ。背景は1:1でくっきり、ドア封鎖の構造原因を断つ。
3. **セーブ/ロード**: 各システムに toJSON/fromJSON、全状態保存、loadGameを「つづきから」に配線。
4. **戦闘解決層**: resolveAction()に式/クリティカル/buff/状態異常を集約、死にメカニクス復活。
5. **歩行方式の決着**: 現行スプライト方式を正とし、旧drawWalkChar/drawSkeletalChar/関連ヘルパを完全削除。
